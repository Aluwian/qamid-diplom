## Границы тестирования
Что проверяем:
- Поля авторизации(валидация полей, пустые поля, пограничные значения, верные данные, 
неверные данные)
- CRUD новостей(создание, редактирование, удаление, фильтрация, сортировка, 
отображение 3х последних новостей на главной странице)
- Навигация (переход по внешним ссылкам на Политика конфиденциальности и Пользовательское соглашение, переход по меню)
- Реакция на прерывание процесса(например свернуть приложение во время 
создания или редактирования новости)

Что не проверяем:
- Точное соответствие пикселей в верстке, отступы и шрифты
- Производительность и нагрузка
- Совместимость с другими версиями Android (тестируется только на API 29)
- Локализация
- Безопасность


## Реализованный функционал
- Авторизация (роль неизвестна, есть доступ ко всему функционалу)
- Сессия (сохранение сессии при закрытии приложения, работа кнопки log out)
- Главный экран
- Лента новостей
- CRUD новости
- Цитаты
- Раздел "О приложении"
- Навигация меню

## Типы тестирования
- Функциональное тестирование (позитивные сценарии, негативные сценарии)
- Проверка состояний(например свернуть приложение во время
  создания или редактирования новости)
- UI-проверка (ввод слишком длинного текста, активность кнопок(при первом входе была неактивна О приложении))

## Предусловия
- Устройство на Android API 29 с установленным приложением
- Интернет соединение
- Для проверки авторизации(загрузка приложения, где вход не осуществлен, либо первая загрузка приложения)
- Для тестов новостей, раздела about, или раздела с цитатами необходима авторизация

## Данные для тестов:
Позитивные сценарии:
- Корректный логин и пароль
- Название, описание новости
- Валидные данные для проверки полей
- Данные для проверки пограничных значений
Негативные сценарии:
- Некорректный логин и пароль
- Данные для проверки Превышение максимальных и минимальных значений

## Способы фиксации результатов
- Allure - файл allure-results.zip с отчетом результата с прогоном тестов
- Check.csv - отметки о пройденных или непройденных тестах;
- Cases.csv - подробное описание тест-кесов и 
отметки о покрытии с указанием причин, почему тест-кейс не может быть покрыт
- Result.md - отчет о сравнении затрат времени на проверку приложения вручную и при помощи UI-тестов


## План 
1. Проверка приложения (7 дней)
- Составление плана тестирования
- Составление чек-листов и тест-кейсов
- Отправка отчета руководителю
2. Автоматизация(2-3 недели)
- Предварительная настройка проекта
- Написание автотестов
- Отправка руководителю на проверку
3. Подготовка отчетности (1 неделя)
- Подключение Allure
- Формирование отчетов
- Финальная сдача проекта

| Инструмент | Назначение | Обоснование выбора                                                                                                                                                                       |
|-----------|-----------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Android Studio + эмулятор Pixel 4 API 29** | Запуск и исследование приложения | Версия API выбрана согласно ТЗ. Эмулятор использован вместо физического устройства, т.к. нет устройства, подходящего под параметры ТЗ                                                    |
| **Logcat** | Анализ системных и прикладных логов приложения, включая HTTP-запросы и ответы | Встроен в Android Studio. Благодаря ему был обнаружен баг: новости успешно создаются, но не отображаются в списке из-за некорректного формата `publishDate` (секунды вместо миллисекунд) |
| **Appium Inspector** | Инспекция UI-элементов, поиск локаторов | Нужен для подготовки к автоматизации — показывает точные идентификаторы объектов, которые затем будут использоваться в Page Object'ах                                                    |
| **Git + GitHub (ветка `draft`)** | Версионирование кода и документации | Стандарт индустрии. Работа в черновой ветке `draft` позволяет разрабатывать автотесты без риска сломать основную ветку                                                                   |
